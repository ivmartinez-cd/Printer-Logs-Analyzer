📌 VISIÓN GENERAL DE LA APP

Una app web profesional interna que:

• analiza logs HP
• detecta incidentes automáticamente
• clasifica severidad
• recomienda acción
• permite ajustar reglas

👉 orientada a uso operativo real.

📌 FLUJO PRINCIPAL (USUARIO)
🔹 Flujo completo

1️⃣ pegar log
2️⃣ seleccionar modelo
3️⃣ validar códigos
4️⃣ analizar
5️⃣ ver dashboard
6️⃣ editar reglas si hace falta

👉 todo en una sola experiencia.

📌 PANTALLA 1 — INPUT

Debe tener:

✔ Área pegar log

• multiline
• soporta TAB
• soporta logs largos

✔ Selector modelo (obligatorio)

👉 si no selecciona → no analiza.

✔ Validación previa (automática)

Mostrar:

• total líneas
• códigos detectados
• advertencia códigos nuevos

👉 si hay código nuevo:

bloquea análisis
abre formulario.

📌 PANTALLA 2 — DASHBOARD PRINCIPAL
🔹 RESUMEN SUPERIOR (KPIs)

Debe mostrar:

• total incidentes
• severidad máxima
• códigos únicos
• eventos en recency window

👉 colores:

• rojo = crítico
• amarillo = monitorear
• azul = informativo

🔹 TIMELINE VISUAL

Debe mostrar:

• línea tiempo horizontal
• eventos dentro/fuera ventana
• colores por severidad

👉 ayuda a entender el patrón.

🔹 TABLA INCIDENTES (CORE)

Columnas:

• código
• clasificación
• cantidad
• ventana
• contador
• recomendación

Debe:

✔ expandirse
✔ mostrar eventos
✔ ordenar
✔ paginar

🔹 TABLA EVENTOS

Debe:

✔ filtrar
✔ ordenar
✔ buscar

Tooltip debe mostrar:

• código
• fecha
• contador
• firmware
• ayuda

📌 FILTROS

Debe permitir:

✔ severidad
✔ código
✔ texto
✔ contador
✔ recency

👉 combinables.

📌 BÚSQUEDA

Debe buscar en:

• código
• ayuda
• descripción

✔ resaltar coincidencias.

📌 EXPORTACIONES

Debe permitir:

✔ PDF completo
✔ CSV completo
✔ CSV solo críticos

👉 listo para Excel.

📌 EDICIÓN REGLAS
🔹 Desde incidente

Debe existir:

👉 botón “editar regla”

🔹 Modal edición

Campos:

• clasificación
• resolución
• X
• Y
• counter_max_jump
• severity_weight
• tags

✔ validación en vivo
✔ confirmación modal
✔ toast feedback

Flujo:

1️⃣ detecta
2️⃣ pide completar
3️⃣ guarda
4️⃣ analiza
